# Concept Writing Rules

> This file is the single source of truth for concept-writing logic. It is read at
> runtime and injected into the generation, evaluation and rewrite prompts. Editing
> it changes how the assistant writes — no code change required.

## 1. Structure of a Concept

A strong concept consists of:
- Consumer Insight
- Benefit
- Reason Why (RTB)
- Claim
- Name

For the MVP, the **required** outputs are:
- Insight
- Benefit
- RTB
- Tagline

**Optional** outputs: Concept Name, Claim, Supporting proof points.

---

## 2. Consumer Insight Rules

A good insight:
- Is written in consumer language
- Reflects real behaviour, needs or tension
- Goes beyond the obvious fact
- Exposes an unmet need
- Is emotionally relevant

Preferred structure (Truth → Need → Friction):
- **Truth:** "I want… / I do…"
- **Need:** "…because…"
- **Friction:** "…but…"

Example: *"I want great colour results because I care how I look, but the chemical smell makes the experience unpleasant."*

---

## 3. Benefit Rules

A benefit:
- Resolves the consumer tension
- Clearly delivers value
- Is easy to understand
- Focuses on the single main outcome

> Rule: the main benefit MUST answer the main insight.

---

## 4. Reason Why (RTB) Rules

An RTB:
- Explains WHY the benefit is believable
- Uses simple "kitchen logic"
- Directly supports the benefit

> Rule: the RTB must NOT introduce a completely new idea that was not set up by the benefit.

---

## 5. Coherence Rules

- A key concept element must reappear across Insight → Benefit → RTB.
- One clear story; no contradictions.
- All elements speak one language and feel like a single thought.

---

## 6. Good Concept Criteria

A good concept is:
- Single-minded (one core message)
- Easy to understand
- Consumer-centric
- Emotionally and functionally relevant
- Concise — short enough to read quickly

---

## 7. Tagline Rules

A tagline:
- Summarises the core idea and/or main selling message
- Is short and memorable
- Reflects the key benefit or experience and the chosen direction

---

## 8. Red Flags (avoid)

- Buzzword overload
- Insights that sound technical instead of human
- A weak link between Insight → Benefit
- Generic benefits ("better", "improved") with no point of difference
- An RTB that pulls in a new, unrelated idea
- A tagline that is catchy but not specific to the concept
